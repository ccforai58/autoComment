# 评论框拟人工逐字输入设计

## 目标

批量提交时，评论内容填入应从“快速赋值”改为更像人工输入的逐字输入。默认使用稳定的人类节奏，同时保留快速和谨慎两种策略，方便后续按站点风控情况切换。

## 策略

- `human-fast`：字符间隔短，适合批量较多的任务。
- `human-normal`：默认策略，字符间隔适中，标点和段落有短暂停顿。
- `human-careful`：更慢，句子和段落停顿更明显。

## 行为要求

- 批量评论默认使用 `human-normal`。
- 每个字符追加后触发 `input` 事件。
- 输入完成后触发 `change` 和 `blur`。
- 保留 href 中真实换行，不清洗用户要求保留的内容。
- 日志记录输入策略、字符数、耗时和平均间隔。

## 验收标准

- 测试证明 `human-normal` 的计划总延迟大于 0。
- 测试证明逐字输入保持 href 换行。
- 批量填充日志出现 `strategy`、`chars`、`durationMs`、`avgDelayMs`。
- `node --check`、相关测试、乱码扫描通过。
