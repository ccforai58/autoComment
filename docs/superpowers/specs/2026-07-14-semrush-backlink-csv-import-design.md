# Semrush 外链 CSV 导入设计

## 背景

当前批量外链提交工作台已经支持普通 CSV 上传，并会根据 CSV 中的 URL 批量打开页面、生成推广评论、提交并记录结果。新的需求是在不影响普通 CSV 流程的前提下，新增一个专门支持 Semrush 外链导出文件的导入入口。

Semrush CSV 中的 `Source url` 是竞争对手已经获得外链的来源页面，本功能只把 `Source url` 作为待提交页面。`Target url` 代表竞争对手网站，不用于校验当前插件里配置的推广网站。

## 目标

- 在批量工作台中新增 Semrush 外链 CSV 导入入口。
- 导入时过滤重复、低质量、站群或有害来源。
- 过滤逻辑严格但可解释，尽量避免误杀。
- 导入完成后给用户清晰统计：解析多少、可提交多少、失败多少、过滤多少、待确认多少。
- 增加详细可诊断日志，方便后续调整过滤规则和排查误判。

## 非目标

- 不展示过滤明细列表。
- 不提供过滤报告导出。
- 不使用 Semrush CSV 的 `Target url` 校验推广网站。
- 暂不使用 `Lost link=TRUE` 作为过滤条件。
- 暂不把 `External links` 作为硬过滤条件。
- 不在导入阶段对所有网站逐个联网检测，只对疑似站群或有害域名做联网复查。

## UI 设计

在现有“上传 CSV 文件”卡片中放置两个并排入口：

- 普通 CSV：保留现有上传能力和提示。
- Semrush 外链 CSV：新增入口，专门处理 Semrush 导出的 backlinks CSV。

Semrush 入口下方提供少量就地配置：

- `Page ascore 最低值`：默认 `5`。
- `同域最多保留`：默认 `3`。

Semrush 导入过程中显示阶段状态：

- 正在解析 CSV
- 正在去重
- 正在进行离线过滤
- 正在联网复查疑似站群
- 导入完成

导入完成后，在文件信息区域显示摘要：

- CSV 总行数
- 解析成功行数
- 可提交 URL 数
- 无效 URL 数
- 重复 URL 数
- 低 AScore 过滤数
- 站群/有害域名过滤数
- 联网复查失败并设为待确认的 URL 数
- 同域超额过滤数

待确认 URL 不进入本次提交队列，也不消耗积分。

## 数据流

1. 用户选择 Semrush 外链 CSV。
2. 前端使用 PapaParse 解析 CSV。
3. 识别以下字段：
   - `Source url`：必需，作为待提交页面。
   - `Page ascore`：可选，用于 AScore 阈值过滤。
   - `Source title`：可选，用于站群离线判断。
   - `External links`：可选，用于同域排序参考和日志。
   - `Last seen`：可选，用于同域排序参考。
4. 逐行规范化 `Source url`。
5. 无效 URL 计入失败。
6. 按规范化 URL 去重。
7. 根据 AScore 阈值过滤，默认低于 `5` 过滤；如果缺少 `Page ascore` 列，则跳过 AScore 过滤并记录日志。
8. 使用离线规则识别疑似站群或有害域名。
9. 对疑似域名去重后，每个域名只联网复查一次。
10. 根据域名复查结果处理同域所有记录：
    - 确认站群或有害：同域全部过滤。
    - 复查通过：同域继续进入候选池。
    - 复查失败或超时：同域全部标记为待确认，不提交。
11. 对候选记录按域名分组，每个域名最多保留 3 条。
12. 同域保留排序：
    - `Page ascore` 高优先。
    - `External links` 少优先。
    - `Last seen` 新优先。
13. 最终可提交 URL 进入现有批量提交队列。

## 过滤规则

### URL 去重

按现有规范化方式去重：

- 补齐协议。
- host 转小写。
- 去掉 hash。
- 去掉末尾多余斜杠。

重复 URL 只保留第一条进入后续过滤。

### AScore 过滤

默认 `Page ascore < 5` 过滤。阈值由用户在 Semrush 导入区配置。

如果 `Page ascore` 为空或无法解析为数字，该行不因 AScore 单独过滤，但应记录到诊断日志的字段异常统计中。

### Lost Link

`Lost link=TRUE` 暂不参与过滤。

### External Links

`External links` 暂不作为硬过滤条件，避免误杀优质博客页。

该字段只用于：

- 同域超过保留上限时的排序参考。
- 诊断日志摘要。

### 疑似站群或有害域名

导入时先进行离线判断。离线判断应保持可解释，可基于以下信号组合：

- 同域记录数量异常多。
- 同域页面标题高度重复。
- URL 路径结构高度重复。
- 域名或路径命中现有非法站点过滤词。
- 域名模式疑似批量生成或低质量目录。

离线规则命中后，不直接整域过滤，而是进入联网复查。

### 联网复查

联网复查按域名去重执行：

- 同一个域名只请求一次。
- 结果缓存并应用到同域全部记录。
- 日志按域名记录，不按每条 URL 重复记录。

复查结果：

- 确认站群或有害：同域全部过滤。
- 复查通过：同域继续筛选。
- 复查失败、超时或网络错误：同域全部标记为待确认，不提交；统计中说明“联网复查失败”。

联网复查需要设置超时，避免大 CSV 导入长时间卡住。

### 同域最多保留

非站群且通过质量判断的域名，默认最多保留 3 条。

同域超过 3 条时，按以下顺序保留：

1. `Page ascore` 高优先。
2. `External links` 少优先。
3. `Last seen` 新优先。
4. 如果仍相同，保持 CSV 原始顺序。

## 统计模型

Semrush 导入应维护结构化统计对象：

- `totalRows`
- `parsedRows`
- `validUrlRows`
- `invalidUrlRows`
- `duplicateUrlRows`
- `lowAscoreRows`
- `suspectedNetworkDomains`
- `confirmedNetworkDomains`
- `networkFilteredRows`
- `networkReviewRows`
- `domainLimitFilteredRows`
- `submitReadyRows`

用户界面只显示摘要，不展示明细。

## 日志设计

导入过程必须输出结构化、可搜索、非敏感日志。建议统一前缀：

- `[batch][semrush-import]`

关键日志点：

- 导入开始：文件名、文件大小、导入模式。
- 字段识别：识别到的列名和索引，缺失字段。
- 用户配置：AScore 阈值、同域保留上限。
- 解析统计：总行数、有效 URL 数、无效 URL 数。
- 去重统计：重复 URL 数。
- AScore 过滤统计：过滤数量、阈值、少量样本。
- 疑似站群离线判断：域名、命中规则、影响行数、少量样本 URL。
- 联网复查开始：域名数量。
- 联网复查结果：域名、状态码、耗时、结论、错误信息。
- 复查缓存命中：同域多 URL 复用同一个结论。
- 同域保留统计：域名、候选数、保留数、过滤数。
- 导入完成：完整统计对象。

日志不得包含账号、密码、cookie、token、私密 URL 参数或其他敏感信息。URL 如包含敏感 query，应在日志中脱敏 query 或只记录 origin + pathname。

## 错误处理

- 缺少 `Source url`：阻止导入，提示文件不是有效 Semrush 外链 CSV。
- 缺少 `Page ascore`：允许导入，提示跳过 AScore 过滤，并写日志。
- CSV 解析失败：提示解析失败，日志记录错误 message。
- 联网复查失败：相关域名标记为待确认，不提交。
- 联网复查超时：同失败处理。
- Semrush 导入异常：不影响普通 CSV 导入入口。

## 测试计划

新增纯逻辑测试，覆盖：

- Semrush 字段识别。
- `Source url` 规范化与去重。
- AScore 默认阈值 5。
- AScore 阈值可配置。
- 缺少 `Page ascore` 时跳过过滤。
- `Lost link=TRUE` 不过滤。
- `External links` 不硬过滤。
- 同域最多保留 3 条。
- 同域排序：AScore 高、External links 少、Last seen 新。
- 疑似站群域名只联网复查一次。
- 同域复查结果缓存并应用到所有记录。
- 复查失败进入待确认且不进入提交队列。
- 导入统计数量准确。

回归测试：

- 现有普通 CSV 导入仍可用。
- 现有批量提交流程不回归。
- 运行 `npm test`。

## 验收标准

- 用户可以在批量工作台看到并排的普通 CSV 和 Semrush CSV 两个入口。
- Semrush 样例 CSV 能被识别，并使用 `Source url` 生成待提交队列。
- 导入统计同时显示解析规模和最终可提交规模。
- 低于默认 AScore 5 的记录会被过滤。
- `Lost link=TRUE` 不会被过滤。
- `External links` 高不会被单独过滤。
- 疑似站群域名只联网复查一次。
- 联网复查失败的同域 URL 显示为待确认，不进入提交队列。
- 日志能说明每类过滤和待确认的原因。
