# 插件工作台易用性重设计

## 背景

插件当前点击图标打开设置页，用户需要再从设置页进入批量处理页。对于本地免费运行定位，最常用路径应是“检查推广配置 -> 上传 CSV -> 启动批量提交 -> 查看当前结果和历史归档”。设置页、支付页和隐私页不应成为主工作流阻碍。

## 目标

- 点击插件图标直接打开 `batch.html` 批量工作台。
- 批量页顶部提供清晰的配置状态、入口导航和任务启动路径。
- 历史归档表增加 CSV 序号列，并按同一批次的 CSV 原始顺序排列。
- 保留设置页作为辅助配置页，方便修改推广网站、联系人和本地用户。
- 本次不修改提交核心逻辑。

## 主工作流

1. 用户点击插件图标，进入批量提交工作台。
2. 工作台顶部显示当前推广网站、姓名/邮箱配置状态和本地用户状态。
3. 如果缺少必要配置，用户可以从工作台直接打开设置页。
4. 用户上传 CSV，确认 URL 列表和拦截情况。
5. 用户启动批量任务，查看进度、当前批次结果和历史归档。

## UI 设计

### 批量工作台

- 标题改为“外链提交工作台”。
- 副标题强调本地运行、CSV 批量提交、结果验证。
- 顶部新增“运行前检查”卡片：
  - 推广网站
  - 姓名/邮箱
  - 本地用户
  - 快捷按钮：打开设置、刷新配置
- 页面顺序调整为：
  1. 运行前检查
  2. 上传 CSV
  3. 自动操作和超时设置
  4. 启动/停止按钮
  5. 处理进度
  6. 提交结果

### 历史归档

- 表格第一列增加 `#`。
- 序号使用 `originalIndex + 1`，对应原 CSV 行顺序。
- 展示排序规则：
  - 同一批次内按 `originalIndex` 升序。
  - 不同批次按批次最近更新时间倒序。
- 若记录没有 `originalIndex`，显示 `--`，并排在同批次末尾。

### 设置页

- 作为辅助配置页保留。
- 保留打开批量处理按钮。
- 不再作为插件图标默认入口。

## 验收标准

- 点击插件图标打开 `batch.html`。
- 批量页顶部能看到当前关键配置状态。
- 历史归档表有序号列。
- 历史归档同批次记录按 CSV 原顺序显示。
- 当前批次结果和历史归档仍可筛选。
- 修改同步到 `dist/auto-comment-plugin` 和 `load-this-extension`。
- `node --check`、乱码扫描和测试通过。
