# 设置返回与历史归档默认筛选设计

## 目标

优化配置页到批量工作台的返回路径，并让提交结果默认聚焦当前批次。历史归档不再默认混合展示所有推广网站，而是默认展示最近任务对应的推广网站。

## 设计

- 配置页 `Open Batch Processor` 改为 `返回工作台`。
- 点击后在当前标签页跳转到 `batch.html`，不再新开标签页。
- 批量工作台在页面显示或重新获得焦点时刷新运行前检查配置。
- 提交结果默认 Tab 固定为“当前批次”。
- 当前批次没有结果时显示空状态提示。
- 历史归档的推广网站下拉默认选择最近归档记录对应的网站。
- 历史归档保留“全部推广网站”选项，但不默认选中。
- 配置页删除 Contact 区域。

## 验收标准

- 设置页按钮在当前标签页返回 `batch.html`。
- 返回工作台后运行前检查自动显示最新配置。
- 提交结果默认显示“当前批次”。
- 无当前结果时不会自动跳到历史归档。
- 历史归档默认按最近推广网站过滤。
- 配置页不再显示 Contact。
- 同步 `dist/auto-comment-plugin` 和 `load-this-extension`。
